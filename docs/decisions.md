# Architecture Decisions

## 1. WXT over plain Vite

WXT provides file-based entrypoints, auto-manifest generation, HMR for extensions, and `allFrames` support out of the box. It wraps Vite internally, so we still get Tailwind v4 via `@tailwindcss/vite` plugin. We use `srcDir: 'src'` to keep source separate from config.

## 2. Standalone tsconfig.json (not extending .wxt/tsconfig.json)

WXT docs recommend `"extends": ".wxt/tsconfig.json"`, but `wxt build` deletes and regenerates `.wxt/` at build start, causing Vite to fail when it reads the tsconfig mid-build. We use a standalone tsconfig with the same compiler options instead.

## 3. Shadow DOM for hint overlays (not page DOM)

All hint overlays, layer indicator, and toast are rendered inside a Shadow DOM container attached to `document.documentElement`. This prevents page CSS from affecting hint styling and prevents our styles from leaking into the page. The host element uses `aria-hidden="true"` and `pointer-events: none`.

## 4. Inline styles inside Shadow DOM (not Tailwind)

Tailwind classes don't work inside Shadow DOM because the stylesheet isn't injected there. All hint overlay styling uses inline styles via `element.style`.

## 5. Union-find for overlap grouping

Overlap detection uses rbush R-tree for spatial indexing, then union-find to group intersecting rects into stacks. This is O(n log n) for the R-tree queries and near-O(n) for union-find with path compression. Each stack is sorted by z-index (desc) then DOM order (asc).

## 6. Prefix-free label generation via tree expansion

Labels are generated by starting with single-char labels (one per charset char), then expanding the shortest label into `k` children when more labels are needed. This guarantees the prefix-free property because only leaf nodes become labels. Net gain per expansion: `k - 1` labels.

## 7. Module-level mutable state in hint-mode and hint-renderer

Content scripts run once per page load in an isolated world. Module-level `let` state is acceptable here since there's exactly one instance per frame. The state machine uses a discriminated union (`inactive` | `active`) to make impossible states unrepresentable.

## 8. Settings via chrome.storage.sync with live updates

Settings are stored in `chrome.storage.sync` under a single `"settings"` key. The content script loads settings on init and watches for changes via `browser.storage.onChanged`, so updates from the options page apply immediately without page reload.

## 9. tabbable library for element scanning

We use the `tabbable` library with its `getShadowRoot` option to scan interactive elements, including those inside open shadow roots. This handles focus-related edge cases (disabled, hidden, tabindex=-1) that manual `querySelectorAll` would miss.

## 10. Content script `allFrames: true` for iframe support

Setting `allFrames: true` in the content script definition makes WXT inject the script into every frame on the page. Each frame runs its own independent hint session with its own overlays.
